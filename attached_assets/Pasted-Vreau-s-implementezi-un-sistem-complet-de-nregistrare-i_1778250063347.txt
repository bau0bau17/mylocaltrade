Vreau să implementezi un sistem complet de înregistrare și verificare pentru Traderi în aplicația MyLocalTrade.

MyLocalTrade este o platformă care conectează clienți cu business owners / tradesmen, cum ar fi plumbers, electricians, builders, roofers, gardeners, gas engineers etc.

Scopul este ca un Trader să se poată înregistra rapid, dar profilul lui să NU fie public și să NU poată primi lead-uri până când nu trece prin pașii de verificare necesari.

Te rog implementează totul pas cu pas, păstrând designul actual dark/mobile-first din aplicație.

1. Schimbă logica generală pentru Trader signup

Înregistrarea Traderului trebuie să fie împărțită în mai multe etape:

Step 1: Basic Trader Account
Step 2: Email and Phone Verification
Step 3: Business Profile Completion
Step 4: Documents and Qualification Upload
Step 5: Admin Review
Step 6: Subscription / Payment
Step 7: Profile goes live only after approval

Important: după prima înregistrare, contul poate exista în sistem, dar profilul NU trebuie să fie vizibil public până când statusul este VERIFIED și subscription-ul este activ.

5. Implementează verificare telefon

După ce emailul este verificat, Traderul trebuie să verifice telefonul prin cod OTP.

Poate fi implementat inițial mock/local dacă nu există încă integrare SMS reală, dar structura trebuie să fie pregătită pentru Twilio sau alt SMS provider.

Flow:

Dacă este corect, phone_verified devine true
Statusul devine PROFILE_INCOMPLETE

14. Public profile visibility rules

Un Trader poate apărea public doar dacă TOATE condițiile sunt îndeplinite:

email_verified = true
phone_verified = true
business_profile_completed = true
required_documents_uploaded = true
admin status = VERIFIED
subscription_status = ACTIVE
account_status is not SUSPENDED
insurance is not expired, where required

Dacă una dintre aceste condiții nu este îndeplinită, profilul trebuie ascuns din:

Search results
Category pages
Public trader profile pages
Recommended traders
Featured traders
Homepage cards

24. Important business rule

A trader account can exist before verification, but a trader profile must not go live until the trader is fully verified and has an active subscription.

This rule must be enforced in the backend, not only in the frontend.

25. Admin notes and rejection reason

Traderul trebuie să primească notificare și să poată încărca document nou.
