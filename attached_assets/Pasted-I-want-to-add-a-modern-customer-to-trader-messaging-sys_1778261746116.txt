Core concept:

When a customer sends an enquiry to a trader, the system should create a conversation between that customer and that trader.

The conversation should be linked to:

* customer user ID, if the customer has an account
* trader user ID / trader profile ID
* enquiry ID
* job category
* postcode / location area
* created date
* latest message date
* status

Conversation status:

OPEN
AWAITING_TRADER_REPLY
AWAITING_CUSTOMER_REPLY
CLOSED
BLOCKED
REPORTED

Message fields:

* conversation_id
* sender_user_id
* sender_role: CUSTOMER / TRADER / ADMIN
* message body
* created_at
* read_at
* edited_at, optional
* deleted_at, optional
* attachment_url, optional for later
* system_message boolean

Customer features:

* Customer can send initial enquiry message
* Customer can view conversation history
* Customer can reply to trader
* Customer can see unread messages
* Customer can close conversation
* Customer can report conversation/trader

Trader features:

* Trader can see all conversations in trader dashboard
* Trader can reply to customer
* Trader can mark conversation as contacted / quoted / completed
* Trader can see unread messages
* Trader can close conversation
* Trader can report abusive customer

Admin features:

* Admin can view reported conversations

* Admin can view messages only when a conversation is reported or for moderation/support purposes
* Admin can block/suspend users if abuse is found
* Admin actions should be logged in audit log
